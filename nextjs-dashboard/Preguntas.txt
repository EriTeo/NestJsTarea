1. ¿Qué es Next.js y por qué se dice que es el framework de React más popular?
Next.js es un framework de React que facilita crear aplicaciones web rápidas y optimizadas, por eso es uno de los más populares.

2. ¿Cuál es el comando exacto con npx para inicializar un proyecto de Next.js asegurando usar la última versión?
npx create-next-app@latest

3. ¿Cuál es el comando utilizado para levantar el entorno de desarrollo y en qué puerto se ejecuta por defecto?
npm run dev. Se ejecuta por defecto en http://localhost:3000

4. ¿Qué función cumple el comando npm run build en un proyecto de Next.js?
Compila la aplicación para producción, optimizando el código, generando archivos estáticos y mejorando el rendimiento.

5. Describe para qué sirven las carpetas public y app
public: almacena archivos estáticos (imágenes, íconos, etc.). app: contiene la lógica principal, páginas, layouts y rutas de la aplicación.

6. ¿Qué es un CSS Module y cómo se usan?
Es un archivo CSS con estilos locales para un componente. Se usan importándolos en el componente, por ejemplo: import styles from './estilo.module.css';

7. ¿Qué hace el componente layout.tsx?
Define la estructura general de las páginas (como navbar o sidebar) y envuelve el contenido (children).

8. ¿Qué es el enrutamiento?
Es el sistema que permite navegar entre diferentes páginas dentro de una aplicación web.

9. ¿Cuál es la diferencia entre Image e img?
img es una etiqueta HTML normal. Image es un componente de Next.js optimizado (mejor carga, lazy load, tamaño automático).

10. ¿Qué es el Lazy Load?
Es una técnica donde los recursos se cargan solo cuando son visibles en pantalla.

11. ¿Funcionamiento del código (imágenes)?
Las imágenes se muestran cuando entran en el viewport, no todas cargan al inicio.

12. ¿Cómo funciona el enrutamiento en Next.js?
Se basa en la estructura de carpetas dentro de app. Cada carpeta representa una ruta.

13. ¿Qué es una ruta en el contexto web y dónde las vimos?
Es una URL que lleva a una página específica. Ejemplo: /dashboard, /dashboard/invoices.

14. ¿Qué arrojó localhost:3000/dashboard a MiduDev?
Mostró errores de renderizado o componentes faltantes.

15. ¿Dónde vimos esos errores en clase?
En la terminal y en el navegador.

16. ¿Qué es el File System Routing?
Es el sistema donde las rutas se crean automáticamente según la estructura de carpetas.

17. ¿Cómo funcionan las rutas anidadas?
Son rutas dentro de otras rutas, por ejemplo: /dashboard/invoices.

18. ¿Por qué es necesario esto?
Para organizar mejor el proyecto y facilitar la navegación en aplicaciones grandes.

19. ¿Para qué sirve el componente Link?
Permite navegar entre páginas sin recargar la aplicación.

20. ¿Qué se carga con <a> y qué con <Link>?
<a> recarga la página completa. <Link> navega sin recargar (SPA).

21. ¿Qué es un Hook?
Es una función especial de React que permite usar estado y otras características en componentes.

22. ¿Qué hooks usa Midu?
useState, useEffect, usePathname.

23. ¿Por qué ocurre el error?
Por usar componentes o hooks sin importar correctamente o sin 'use client'.

24. ¿Es posible ejecutar componentes de React en cliente y servidor?
Sí, Next.js permite ambos.

25. ¿Qué es el Code Splitting?
Es una técnica que divide el código en partes para cargar solo lo necesario y mejorar el rendimiento.
